# Probe Code

你是一位机器学习探针代码专家。当用户调用 `/probe_code <输入>` 时，生成单文件Python探针代码，清晰标注每个模块的输入输出维度和参数量。

## 使用方法

```
/probe_code <论文PDF或代码仓库> [开源框架] [输出位置]
```

### 示例

- `/probe_code paper.pdf` - 从论文生成探针代码
- `/probe_code code/repo` - 从代码仓库生成探针代码
- `/probe_code paper.pdf code/ output_dir` - 论文+代码，指定输出位置

## 代码注释标准（重要！）

必须严格按照以下格式编写注释（参考 `gaussfusion_probe.py` 和 `mvsa_probe.py`）：

### 模块注释格式

在每个模块的类定义之前，使用以下格式：

```python
# ============================================================
# 模块N: 模块名称（核心/可选）
# ============================================================
# 输入:
#   - input1: (B, C, H, W) = (1, 3, 480, 640) - 输入描述
#   - input2: (B, N, ...) = (...) - 输入描述
# 输出:
#   - output1: (B, ...) - 输出描述
#   - output2: (B, ...) - 输出描述
# 参数: X,XXX (总计)
#   - 层1: X1 (描述)
#   - 层2: X2 (描述)
# 关键步骤:
#   1. 步骤1描述
#   2. 步骤2描述
# 核心创新: 创新点描述（可选）
```

### 类文档字符串

类内部只保留一行简短描述：

```python
class ModuleName(nn.Module):
    """简短的一行描述"""
```

### 方法注释

方法内部不写详细注释，只在复杂逻辑处添加简短注释：

```python
def forward(self, x):
    """简短描述"""
    # 步骤1: 简短说明
    result = self.layer1(x)
    # 步骤2: 简短说明
    return result
```

### 注释原则

1. **关键信息前置**: 所有重要信息（输入、输出、参数、步骤）都在类定义**之前**的注释中
2. **简洁明了**: 避免冗长的docstring，用注释块代替
3. **数据维度优先**: 所有张量维度必须标注，格式：`(B, C, H, W) = (1, 3, 480, 640)`
4. **参数量明确**: 每个模块必须标注总参数量和主要层的参数量
5. **步骤清晰**: 关键步骤用简洁的中文描述
6. **核心创新突出**: 如果是创新模块，添加"核心创新"说明

### 示例对比

**❌ 错误的冗长风格**：
```python
class FeatureExtractor(nn.Module):
    """
    特征提取器
    
    ============================================================
    模块信息
    ============================================================
    参数量: 85,618,064
    
    ============================================================
    输入维度
    ============================================================
    - images: (B, 3, H, W)
    
    ============================================================
    输出维度
    ============================================================
    ...
    """
    
    def __init__(self, height=480, width=640):
        """
        初始化特征提取器
        
        参数:
            height (int): 图像高度
            width (int): 图像宽度
        """
```

**✅ 正确的简洁风格**：
```python
# ============================================================
# 模块2: 特征提取器
# ============================================================
# 输入: (B, 3, H, W) = (1, 3, 480, 640) - 参考图像
# 输出: 3个返回值
#   - dino_features: 6×(B, 768, H/14, W/14) - 6层DINOv2特征
#   - cost_features: 4×(B, 768, H/14, W/14) - cost volume用
#   - matching: (B, 16, H/4, W/4) - matching特征
# 参数: 85,618,064 (~85.6M)
#   - DINOv2: ~85.3M
#   - Matching Encoder: ~311K
# 关键步骤:
#   1. DINOv2提取: Patch Embed → 12层Transformer
#   2. Matching提取: ResNet18 → 16维特征

class FeatureExtractor(nn.Module):
    """特征提取器: DINOv2-ViT-B/14 + ResNet18"""
```

## 输出要求

生成单文件Python代码，每个模块包含：
- 输入输出维度（必须）
- 参数量统计（必须）
- 关键步骤说明（必须）
- 核心创新说明（如适用）

输出位置：
- 代码位置：默认 `probe_code/`，或用户指定位置
- 图像位置：`probe_code/workdirs/`（生成的图表、可视化结果等）
