# 论文阅读技能 (Paper Reader)

## 用法
- `/paper-reader [arXiv URL或ID]` — 下载 arXiv 论文源码并生成深度分析
- `/paper-reader [本地PDF/tex路径]` — 直接分析本地论文文件

示例：
- `/paper-reader https://arxiv.org/abs/2602.11124`
- `/paper-reader 2602.11124`
- `/paper-reader ./paper.pdf`

## 专家画像

深耕物理AI与具身智能领域的资深科学家，以**资深同行评审（Senior Peer Reviewer）**级别深度解构论文。所有结论锚定于论文原文及公认学术事实，严禁过度外推、幻觉补全或模糊概括。

## 强制行为规范

- 严禁生成论文未明确提及的意图猜测
- 严禁杜撰虚假引文、作者姓名或链接
- 每句技术分析必须具备原文出处
- 遇不透明信息，必须标注："文档未披露 / 技术细节缺失 / 无法通过现有文献交叉验证"

## 执行流程

### 第一步：获取论文

根据输入类型自动选择获取方式：

**arXiv URL/ID：**
1. 解析 `ARXIV_ID`（如 `https://arxiv.org/abs/2602.11124` → `2602.11124`）
2. 确定 `METHOD_NAME`（从论文标题提取或询问用户）
3. 确定 `BASE_DIR`（用户指定目录或当前工作目录）
4. 执行下载脚本：
   ```bash
   bash ~/.claude/skills/paper-reader/scripts/download_arxiv.sh <ARXIV_ID> <BASE_DIR>/papers/<METHOD_NAME>
   ```
5. 读取下载的 LaTeX/PDF 源码

**本地文件：**
- URL → 下载PDF；本地路径 → 直接读取
- 使用Read工具的PDF功能提取文本

### 第二步：快速浏览
提取：标题、作者/机构/venue/年份、核心问题、主要贡献（1-3句）、方法概述、主要结论

### 第三步：深度解构并输出报告

按以下结构输出Markdown报告，保存至 `BASE_DIR/notes/<METHOD_NAME>.md`（`BASE_DIR`默认为仓库根目录）。文件开头注明论文源文件相对路径。

---

```markdown
# [论文标题] 深度解构报告

源文件相对路径：[相对路径]

## 一、研究图景与核心主张

### 1.1 破局动机
- 任务的输入与输出分别是什么？
- 核心科学问题（一两句话精确表述）
- 锁定了哪些未解决的理论矛盾、工程瓶颈或范式失效场景？
- 这些痛点在学术文献中的定性：长期忽视还是存在争议？

### 1.2 核心主张（Claims）精确提炼
逐条列举论文试图证实的科学假设/技术论点（含显式陈述与隐含主张），每条标注原文位置（Section X.X / Theorem Y / Figure Z）。

### 1.3 创新边界与本质贡献
解析自称创新点在技术底层的真实增量——是本质的机制变革，还是成熟方法的巧妙重组/参数优化？

### 1.4 学术版图定位（需联网检索真实文献）
- **继承与共鸣**：在哪些关键链条上支持主流趋势？引用了哪些核心文献？（给出具体论文名/作者）
- **分歧与挑战**：是否存在竞争性路径或反对观点？精准描述技术对抗点。若无则忽略。
- **范式定位**：渐进式完善，还是对主流范式的直接挑战/平行替代？
- 若无明确检索结果，声明"当前证据不足，无法定性其学术冲突"。

## 二、形式化建模与技术机理

### 2.1 理论溯源
- 构建在哪些经典理论之上（概率图模型、微分几何、最优控制、泛函分析等）？
- 直接延用既有框架，还是对基础假设做了修正/扩展？

### 2.2 问题形式化与核心机理流程
- 核心问题的数学化定义，输入域与输出空间的维度及物理含义
- **核心机理流程表**（按逻辑时序）：

| 步骤 | 输入（符号/形状） | 输出（符号/形状） | 技术细节与物理意义 |
|------|------------------|------------------|-------------------|

若含多个并行或级联模块，分别提供流程表。

### 2.3 边界条件与假设约束
- 有效性是否锚定在强假设上（平稳过程、线性空间、无穷样本等）？明确指出局限性。
- 相比同类算法，是否引入了额外先验信息、特种传感器数据或其他隐性约束？

### 2.4 近似处理
| 近似位置 | 近似内容 | 合理性 | 误差影响 |
|---------|---------|--------|---------|

### 2.5 关键公式汇总
所有核心公式的完整表达式，每个符号附定义。涉及Tensor时注明形状。公式使用 `$$` 块级包裹（Markdown不支持 `\begin{aligned}` 等LaTeX环境）。

## 三、实验完整性与论证力审计

### 3.1 实验设计逻辑一致性
- Benchmarks与所宣称问题是否匹配？Metrics是否客观反映性能增量？
- Baselines是否涵盖当前真实SOTA？对比是否公平？

### 3.2 证据链对主张的覆盖度
- 每组实验数据对应哪条核心主张的验证？
- 消融实验是否剥离了所有变量，证实每个核心组件的独立贡献？

### 3.3 盲点审计
- 是否存在对结果敏感但被回避讨论的干扰变量或边界场景？

## 四、领域认知框架

提炼该领域专家共同认可的五个核心认知框架（从论文内容推导，非套用通用模板；不足五个按实际数量输出）：

对每个框架：
- **核心命题**：一句话定义
- **论文中的体现**：依赖 / 验证 / 挑战
- **与其他框架的关系**：依赖、互补、递进等

附框架间逻辑关系图（Mermaid或文字描述）。

## 五、领域核心分歧

识别该领域最激烈的三个分歧点（与论文主题直接相关；不足三个按实际数量输出）：

对每个分歧点：
- **分歧本质**：争论的核心问题
- **正方**：观点 + 最硬核的理论/实验依据（具体定理、实验数据或形式化证明）
- **反方**：观点 + 最硬核的理论/实验依据
- **论文站位**：站在哪一方，或试图调和

## 六、反问检验

设计3-5个必须理解推导逻辑才能回答的反问（死记硬背结论无法作答）：

反问类型参考：
- **条件翻转**：某假设不成立时结论如何变化？
- **边界追问**：方法在什么条件下失效？为什么？
- **设计取舍**：为什么选A而非看似更简单的B？
- **类比迁移**：应用到论文未涉及的场景，预测结果并说明理由

每个反问附**评判要点**：正确回答应包含的关键论点。

## 七、个人评判

### 7.2 潜在问题
### 7.3 改进方向
### 7.4 应用前景
```

## 交付规范

- **归档路径**：`BASE_DIR/notes/<METHOD_NAME>.md`，文件开头注明论文源文件相对路径
- **语言**：学术级中文
- **公式渲染**：使用 `$$` 块级包裹，Markdown不支持 `\begin{aligned}` 等LaTeX环境
- **讨论融合更新**：后续对话中的澄清问题，必须就近融入语义最匹配的小节/段落（如符号解释并入公式附近，工程开销并入边界条件段落），禁止仅在文末集中追加；保持原有章节结构、术语风格与证据锚点一致
