# 论文阅读技能 (Paper Reader)

## 用法
`/paper-reader [论文PDF路径或URL]`

## 任务描述
本技能用于深度阅读学术论文，从专家视角提取论文的核心贡献、技术细节和实现方式，帮助用户快速理解论文精髓并掌握关键细节。

## 执行流程

### 第一步：获取论文内容
- 如果提供的是URL，尝试下载PDF
- 如果提供的是本地PDF路径，直接读取
- 提取论文的文本内容（使用Read工具的PDF功能）

### 第二步：快速浏览（第一遍阅读）
**目标**：理解论文的整体框架和核心贡献

提取以下信息：
- **论文标题**：完整标题
- **作者信息**：作者、机构、发表时间、发表 venue
- **核心问题**：论文要解决什么问题？
- **主要贡献**：论文的核心创新点（1-3句话）
- **方法概述**：用了什么方法？
- **实验结果**：主要结论是什么？

### 第三步：精读分析（第二遍阅读）
**目标**：深入理解技术细节和实现方式

#### 3.1 核心观点提取
- **问题定义**：形式化的问题描述
- **核心假设**：论文基于哪些基本假设？
- **关键洞察**：作者的灵感来源和关键发现
- **理论贡献**：新的理论、定理、证明

#### 3.2 建模方法分析
详细分析论文的建模方法：
- **数学模型**：
  - 目标函数：完整的数学表达式
  - 约束条件：所有约束
  - 变量定义：每个符号的含义
  - 推导过程：关键推导步骤

- **算法设计**：
  - 算法流程：用伪代码表示
  - 时间复杂度：分析
  - 空间复杂度：分析
  - 收敛性证明：关键证明思路

- **架构设计**（如涉及深度学习）：
  - 整体架构图：用文字描述或Mermaid图
  - 关键模块：每个模块的作用
  - 创新点：相比现有工作的改进

#### 3.3 实现细节说明
解释每个关键细节：
- **为什么这样设计**：设计动机和理由
- **为什么不能简化**：复杂性的必要性
- **如何实现**：具体的实现技巧
- **关键参数**：重要参数的选择和影响

#### 3.4 近似处理分析
- **哪些地方做了近似**：列出所有近似假设
- **近似的合理性**：为什么可以这样近似
- **近似的影响**：对结果的误差影响
- **如何改进**：可能的改进方向

#### 3.5 局限性分析
- **理论局限**：方法的边界条件
- **实践局限**：实施中的困难
- **适用范围**：什么情况下有效/无效
- **未来工作**：作者提到的未解决问题

#### 4.2 LaTeX格式报告
创建文件：`[论文标题简写]_report.tex`

```latex
\documentclass[12pt,a4paper]{article}
\usepackage[utf8]{inputenc}
\usepackage{amsmath,amsfonts,amssymb}
\usepackage{graphicx}
\usepackage{algorithm}
\usepackage{algorithmic}
\usepackage{hyperref}

\title{[论文标题] \\ 阅读报告}
\author{分析者}
\date{\today}

\begin{document}
\maketitle

\section{基本信息}
\begin{itemize}
    \item \textbf{标题}：[完整标题]
    \item \textbf{作者}：[作者列表]
    \item \textbf{发表}：[Venue] [年份]
\end{itemize}

\section{摘要总结}
[内容...]

\section{研究背景与动机}
[内容...]

\section{核心贡献}
[内容...]

\section{方法详解}
[包含数学公式、算法伪代码等]

\section{实验结果}
[内容...]

\section{个人思考}
[内容...]

\end{document}
```

编译LaTeX文件：
```bash
pdflatex [论文标题简写]_report.tex
bibtex [论文标题简写]_report
pdflatex [论文标题简写]_report.tex
pdflatex [论文标题简写]_report.tex
```

### 第五步：文件组织
创建文件夹：`[论文标题简写]/`
```
[论文标题简写]/
├── [论文标题简写]_report.md      # Markdown报告
├── [论文标题简写]_report.tex     # LaTeX源文件
├── [论文标题简写]_report.pdf     # 编译后的PDF
├── original_paper.pdf            # 原始论文PDF
└── figures/                      # 提取的图表（如果有）
    ├── figure1.png
    ├── figure2.png
    └── ...
```

## 输出要求
- **格式**：同时输出Markdown和LaTeX（已编译的PDF）
- **语言**：中文
- **详细程度**：专家级，每个细节都要解释"为什么"
- **数学公式**：在LaTeX中使用正式的数学环境
- **伪代码**：使用algorithm环境
- **完整性**：必须包含所有核心公式的完整表达式
- **组织**：所有文件放在一个文件夹中

## 质量标准
- [ ] 所有数学公式完整且准确
- [ ] 每个设计细节都有"为什么"的解释
- [ ] 明确列出所有基本假设
- [ ] 分析所有近似处理及其影响
- [ ] LaTeX文件成功编译为PDF
- [ ] 报告结构清晰、层次分明
