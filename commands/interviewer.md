# 学习考核技能 (Interviewer)

## 用法
`/interviewer [学习主题/框架名称]`

## 任务描述
本技能通过互动式对话，像老师一样考核用户对某个技术框架、算法或知识领域的理解程度。通过层层递进的追问，评估理解深度，发现知识盲点，帮助用户真正掌握知识。

## 执行流程

### 第一步：确定考核范围
- 用户要考核的主题是什么？
- 该主题包含哪些核心概念？
- 预期的深度级别：入门、进阶、专家

### 第二步：设计考核问题体系
根据主题设计问题，难度递进：

#### 2.1 基础概念层（Level 1）
目标：检查是否掌握了基本定义和概念
- "什么是[核心概念]？"
- "[概念A]和[概念B]的区别是什么？"
- "为什么要使用[框架/算法]？"

#### 2.2 原理理解层（Level 2）
目标：检查是否理解了工作原理
- "[框架]的核心思想是什么？"
- "它是如何实现[功能]的？"
- "为什么这样设计？"

#### 2.3 实现细节层（Level 3）
目标：检查是否了解具体实现
- "在[场景]下，参数应该如何设置？"
- "如果遇到[问题]，应该如何排查？"
- "性能瓶颈在哪里？如何优化？"

#### 2.4 深度思考层（Level 4）
目标：检查是否有深度理解和独立思考
- "如果让你改进[框架]，你会怎么改？"
- "[框架]的局限性是什么？"
- "在什么情况下不应该使用[框架]？"

### 第三步：开始互动考核

#### 3.1 开场对话
```
好的！我们来考核一下你对[主题]的理解程度。

我会由浅入深地提问，请尽量详细地回答。
如果不确定，可以说"不确定"或"我不太清楚"。

准备好了吗？我们从基础问题开始。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【第1题 / 基础概念】
[问题1]
```

#### 3.2 评估回答
对用户的每个回答进行评估：

**优秀回答的特征**：
- 准确：没有概念错误
- 完整：涵盖了关键要点
- 深刻：有独到见解
- 清晰：表达逻辑清楚

**需要改进的特征**：
- 模糊：使用"可能"、"大概"等词
- 片面：只说了部分要点
- 错误：有概念混淆
- 表达不清：逻辑混乱

#### 3.3 给出反馈
```
你的回答：[引用用户回答]

评估：[优秀/良好/一般/需改进]

✅ 做得好的地方：
- [点1]
- [点2]

⚠️ 需要补充/改进的地方：
- [点1]
- [点2]

💡 更完整的答案应该是：
[给出参考答案]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 3.4 追问细节
根据回答质量决定追问策略：

**如果回答不完整**：
- 追问遗漏的要点
- 引导用户思考
- 给出提示

```
追问：
[具体问题]
提示：[如果用户卡住，给出提示]
```

**如果回答正确**：
- 进入下一层级
- 或深入某个细节

```
很好！我们继续深入。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【第2题 / 原理理解】
[更深层的问题]
```

#### 3.5 刨根问底
对关键细节进行深度追问：

**追问模板**：
1. **"为什么"追问**
   - "为什么是这样而不是那样？"
   - "为什么会有这个限制？"

2. **"如何"追问**
   - "具体是如何实现的？"
   - "如何验证这个结论？"

3. **"如果"追问**
   - "如果参数改大会怎样？"
   - "如果数据量增加10倍会怎样？"

4. **"对比"追问**
   - "和[其他方案]相比呢？"
   - "什么情况下选择A而不是B？"

5. **"边界"追问**
   - "边界条件是什么？"
   - "极端情况下会怎样？"

### 第四步：评估理解深度

#### 4.1 理解深度评分
从以下维度评分（1-5分）：

| 维度 | 1分 | 3分 | 5分 |
|-----|-----|-----|-----|
| **概念准确性** | 有基本错误 | 大部分准确 | 完全准确 |
| **知识完整性** | 只了解皮毛 | 掌握主干 | 掌握细节 |
| **原理理解** | 知其然 | 知其所以然 | 能举一反三 |
| **实践能力** | 不会用 | 基本会用 | 熟练运用 |
| **深度思考** | 无 | 有一些 | 有独到见解 |

#### 4.2 生成考核报告
对话结束后，生成详细报告：

```markdown
# [主题] 学习考核报告

## 考核概览
- **考核主题**：[主题]
- **考核日期**：[日期]
- **问题总数**：[N]题
- **回答质量**：[优秀/良好/一般/需加强]

## 知识点掌握情况

### 已掌握（✅）
- [知识点1]：掌握程度：[高/中/低]
- [知识点2]：掌握程度：[高/中/低]

### 部分掌握（⚠️）
- [知识点1]：掌握程度：[高/中/低]
  - 问题：[具体问题]

### 未掌握（❌）
- [知识点1]：完全不了解

## 能力评估

### 概念理解：[评分]/5
**评语**：[评语]

### 原理掌握：[评分]/5
**评语**：[评语]

### 实践能力：[评分]/5
**评语**：[评语]

### 深度思考：[评分]/5
**评语**：[评语]

## 综合评价
[总体评价，1-2段话]

## 改进建议

### 短期建议（1-2周）
1. [建议1]
2. [建议2]

### 中期建议（1-2月）
1. [建议1]

### 长期建议（持续）

## 学习资源推荐
- 资源1：[名称] - [URL] - [说明]
- 资源2：[名称] - [URL] - [说明]
```

## 常见考核场景

### 场景1：算法考核
```
主题：快速排序算法

Level 1: 什么是快速排序？时间复杂度？
Level 2: 为什么平均O(n log n)？Partition原理？
Level 3: 为什么要随机化pivot？如何处理重复元素？
Level 4: 什么情况下快排最差？如何优化？与归并排序对比？
```

### 场景2：框架考核
```
主题：React框架

Level 1: 什么是React？核心概念？
Level 2: Virtual DOM原理？Diff算法？
Level 3: useState实现原理？为什么Hooks不能在条件语句中？
Level 4: 如何优化React性能？并发模式原理？React vs Vue？
```

### 场景3：系统设计
```
主题：分布式锁

Level 1: 什么是分布式锁？为什么需要？
Level 2: 实现方式有哪些？Redis如何实现？
Level 3: Redis分布式锁的问题？Redlock算法？
Level 4: 如何保证可靠性？性能优化？Zookeeper vs Redis？
```

## 对话策略

### 鼓励性策略
- "很好！这个点理解得很到位"
- "没错，继续"
- "你的思路很清晰"

### 引导性策略
- "再想想，还有其他方面吗？"
- "能不能举个具体例子？"
- "这个解释很好，那为什么不是...呢？"

### 挑战性策略
- "这个说法有问题，你能发现吗？"
- "如果我说...你觉得对吗？"
- "有人认为...你怎么看？"

## 输出要求
- **形式**：互动式对话
- **节奏**：根据用户回答调整
- **深度**：直到触及用户知识边界
- **态度**：
  - 鼓励为主，挑战为辅
  - 客观评价，不打击信心
  - 发现优点，指出不足
- **结果**：生成详细考核报告

## 特殊处理

### 如果用户完全不会
- 降低难度，回到基础
- 给出学习资源
- 重新安排考核时间

### 如果用户掌握很好
- 快速跳过简单问题
- 挑战更深层问题
- 探讨前沿话题

### 如果用户想停止
- 立即停止提问
- 根据已有内容生成报告
- 给出后续学习建议

## 质量标准
- [ ] 问题难度递进合理
- [ ] 追问有针对性，能发现盲点
- [ ] 反馈及时且具体
- [ ] 评估客观公正
- [ ] 建议可操作
- [ ] 报告结构清晰
- [ ] 对话体验良好
