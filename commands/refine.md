# 细节补充与文档完善技能

## 用法
`/refine [文档路径] [问题/关注点]`

**示例**：
```
/refine analysis.md "状态转移矩阵F是如何推导的？"
/refine paper.md "过程噪声协方差Q是调参还是理论推导？"
/refine code.md "这个时间复杂度分析中的n是什么？"
/refine lio_sam.md "特征提取是在点云上还是深度图上？深度图的作用是什么？"
```

## 适用场景
- **代码分析**：对已有分析报告补充关键实现细节
- **论文阅读**：补充论文中略过的推导步骤
- **技术文档**：澄清模糊描述,添加代码证据
- **架构理解**：深入解释为什么这样设计

## 执行流程

### 第一步：读取并定位
1. 读取目标文档 (`.md` 或 `.tex`)
2. 理解文档结构和当前内容
3. 定位与问题相关的章节

### 第二步：深入分析
**自动探索代码/论文寻找答案,不向用户提问**:

#### 分析策略
1. **表层分析** (是什么)
   - 追踪相关代码路径
   - 读取关键函数实现
   - 查找变量定义和使用
   - 定位公式/算法的具体实现

2. **中层分析** (为什么)
   - 分析设计决策的动机
   - 对比可能的实现方案
   - 查找注释和文档说明
   - 理解性能/精度权衡

3. **深层分析** (怎么做)
   - 追踪数据流
   - 分析算法复杂度
   - 理解数学推导
   - 提取核心代码片段

#### 分析工具
- **Grep**: 搜索关键词,定位相关代码
- **Read**: 读取源码文件
- **Glob**: 查找相关文件
- **LSP**: 查找符号定义和引用
- **代码推理**: 理解逻辑流程

### 第三步：整合到文档
将分析结果以结构化方式补充到文档:

#### 更新位置选择
- **优先**: 在相关章节后添加"细节补充"子节
- **备选**: 在文档末尾添加"FAQ"或"深入理解"章节
- **特殊情况**: 如果原有描述不准确,直接修改原文

## 分析维度

### 代码类问题分析维度

#### 1. 算法细节
- **边界条件**: 如何处理特殊情况?
- **数据结构**: 为什么选择这种结构?
- **时间/空间复杂度**: 具体是多少?瓶颈在哪里?
- **数值稳定性**: 如何避免精度损失?

#### 2. 工程实现
- **设计决策**: 为什么A方案而非B方案?
- **性能优化**: 哪些优化技巧?效果如何?
- **错误处理**: 异常情况如何处理?
- **可扩展性**: 如何支持未来扩展?

#### 3. 数学建模
- **公式推导**: 每一项的物理意义?
- **近似假设**: 做了哪些简化?适用范围?
- **参数选择**: 阈值/权重如何确定?
- **误差分析**: 误差来源?如何传播?

### 论文类问题分析维度

#### 1. 理论推导
- **推导依据**: 每一步的数学基础?
- **简化合理性**: 为什么可以这样简化?
- **适用边界**: 理论成立的条件?
- **误差界**: 理论误差上界?

#### 2. 实验设计
- **参数选择**: 为什么这些超参数?
- **数据集特点**: 什么场景下有效/失效?
- **消融实验**: 每个组件的贡献?
- **对比方法**: 与其他方法的本质区别?

#### 3. 方法细节
- **关键步骤**: 论文中略过的实现细节?
- **工程技巧**: 从代码中发现的trick?
- **潜在问题**: 方法可能的局限性?
- **改进方向**: 可以如何优化?

6. 理解完整流程:去畸变(3D) → 投影到深度图 → 计算深度曲率
7. 补充到文档第3.2.2节
```

**文档更新**:
```markdown
### 3.2.2 特征提取

原有内容...

### 细节补充 🆕

**问题**: 特征提取是在点云上还是深度图上?深度图的作用?

**核心发现**:
- 特征提取的**曲率计算**在**深度图的深度值**上进行,而非3D点云坐标
- 深度值存储在 `cloudInfo.pointRange[i]` 中
- 曲率公式: `diffRange = pointRange[i-5] + ... - 10*pointRange[i] + ...`

**代码证据**:
```cpp
// featureExtraction.cpp:86-91
float diffRange = cloudInfo.pointRange[i-5] + ... - 10*cloudInfo.pointRange[i] + ...;
cloudCurvature[i] = diffRange * diffRange;  // 深度曲率,非空间曲率
```

**为什么用深度图?**
1. **高效邻域访问**: 深度图是有序2D矩阵,邻域索引直接确定,复杂度O(1)
   - 3D点云是无序集合,需要KD树查找邻域,复杂度O(log N)
2. **快速计算**: 深度曲率只需1D减法,空间曲率需要3D欧氏距离
3. **保持测距精度**: 深度是激光雷达原始测量,避免坐标转换引入的误差

**去畸变与深度图的关系**:
```
原始3D点 → deskewPoint (3D空间去畸变) → 投影到深度图(rangeMat) → 计算深度曲率
```
- **去畸变在3D空间完成**,使用刚体变换(旋转+平移)
- **深度图是去畸变后的存储格式**,用于加速特征提取

**数据流追踪**:
```cpp
// imageProjection.cpp:565 - 先去畸变
thisPoint = deskewPoint(&thisPoint, laserCloudIn->points[i].time);

// imageProjection.cpp:567 - 存入深度图
rangeMat.at<float>(rowIdn, columnIdn) = range;

// imageProjection.cpp:589 - 提取时保留深度值
cloudInfo.pointRange[count] = rangeMat.at<float>(i,j);
```

## 注意事项

### 分析质量
- ✅ **基于证据**: 所有结论必须有代码/论文证据
- ✅ **深入理解**: 不满足于表面,追问"为什么"
- ✅ **完整追踪**: 追踪数据流和控制流
- ✅ **多角度**: 从算法、工程、数学多维度分析

### 文档更新
- ✅ **结构清晰**: 使用统一的格式模板
- ✅ **易于理解**: 添加代码示例和图表
- ✅ **位置合理**: 补充内容靠近相关章节
- ✅ **可追溯**: 标注更新时间和代码位置

### 避免事项
- ❌ 不要向用户提问,自己找到答案
- ❌ 不要只描述"是什么",要解释"为什么"
- ❌ 不要堆砌代码,提取关键片段
- ❌ 不要过度推理,基于事实分析
